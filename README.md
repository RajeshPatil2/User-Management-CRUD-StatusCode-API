
USER MANAGEMENT CRUD API
(Spring Boot + MySQL + HTTP Status Codes)
==========================================

PROJECT OVERVIEW
----------------
This project is a **Spring Boot REST API**.
It performs **CRUD operations** on User data.
It returns **proper HTTP Status Codes** for each API response.
This project follows **industry-standard backend practices**.

-----------------------------------------
KEY FEATURES
------------------------------------------
**Create User (POST API)**
**Get All Users (GET API)**
**Get Single User by ID**
**Update User by ID**
**Delete User by ID**
**Proper HTTP Status Codes handling**
**Clean Controller-Service-Repository structure**

-----------------------------------------
API ENDPOINTS
-----------------------------------------
POST    /postData          → Add new User  
GET     /getData           → Get all Users  
GET     /getData/{uid}     → Get User by ID  
PUT     /postData/{uid}    → Update User  
DELETE  /postData/{uid}    → Delete User  

------------------------------------------
HTTP STATUS CODES USED
------------------------------------------
**200 OK**       → Data fetched / updated successfully  
**202 ACCEPTED** → Data saved successfully  
**400 BAD REQUEST** → Invalid User ID or data not found
